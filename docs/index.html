NJSQL

Not Just Structured Query Language

High-performance standalone hybrid DBMS in Java. JSON-native storage with full SQL interface, referential integrity, and Git for Data™.

Zero Dependencies ~3100 QPS MIT License Java 23+
Get Started GitHub

Features

Git for Data™

Full version control: push, pull, clone, branch, merge with automatic conflict resolution. Parallel development without lock-in.

SQL-Level Integrity

Enforces PRIMARY KEY, FOREIGN KEY with ON DELETE CASCADE/SET NULL, UNIQUE, NOT NULL. Schema validation at runtime.

Zero-Dependency Server

Standalone microservice — no external DB, no JDBC. Runs anywhere Java 23+ is installed.

Permission-Based Multi-Tenancy

Isolated namespaces per user with granular RBAC. Secure and scalable.

CLI + SQL REPL

Interactive shell with history, multi-line input, and powerful commands.

Human-Readable Storage

All data in .json + _schema.json — inspect, diff, or recover manually.

Performance Proof

Tested on Intel i7-12700H, 16GB RAM, JDK 23, 10K sequential INSERT/SELECT

Layer QPS Latency (p95) Notes
Core Engine ~3100 0.8 ms Direct in-memory + disk sync
REST API (Current) ~500 5.2 ms Bottleneck: HTTP + JSON serialization

Insight: API layer is 6.2× slower due to servlet overhead.
Roadmap: gRPC + streaming JSON to close the gap.

Installation

Requirements

  • Java JDK 23 (or later)
  • Windows / Linux / macOS

Run the Server

# Windows
run.bat

# Linux / macOS
chmod +x run.sh
./run.sh

Server starts at http://localhost:2801 

>> Welcome to NJSQL (Hybrid JSON-SQL DBMS)
>> Type /help for commands. Type /exit to quit.
NJSQL>

SQL Mode

CREATE DATABASE test;
USE test;

CREATE TABLE users (
  id INT PRIMARY KEY,
  name TEXT NOT NULL,
  dept_id INT,
  FOREIGN KEY (dept_id) REFERENCES departments(id) ON DELETE SET NULL
);

CREATE TABLE departments (
  id INT PRIMARY KEY,
  name TEXT UNIQUE
);

INSERT INTO departments VALUES (1, 'Engineering');
INSERT INTO users VALUES (101, 'Alice', 1);

SELECT * FROM users WHERE name LIKE 'A%';
DESCRIBE users;
